# Wave 4 — 真实发布链路设计文档

## 概述

Wave 4 将发布管理从 mock 状态机升级为真实的跨平台发布链路。用户配置好各平台的开发者账号后，可以直接从盎然内容发布到小红书、YouTube、TikTok、Facebook。

## 设计目标

1. 用户配置平台账号的过程清晰安全
2. 发布操作有明确的反馈和进度展示
3. 多平台发布支持逐平台状态跟踪
4. 失败可重试，不阻塞其他平台
5. 等待东闫提供各平台开发者账号后直接对接

## 1. 平台账号连接流程

### 入口

**位置：** Key 管理页面底部或独立「平台连接」页面

```
Key 管理
├── AI Provider Keys (现有)
└── ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
    ┌─ 平台账号 ─────────────────┐
    │  小红书      ○ 未连接       │
    │  YouTube     ○ 未连接       │
    │  TikTok      ○ 未连接       │
    │  Facebook    ○ 未连接       │
    └─────────────────────────────┘
```

### 连接流程

```
[未连接] → 点击「连接」→ 弹出配置面板
                         ├─ OAuth 方式：跳转授权 → 回调 → 保存 token
                         └─ Cookie/手动方式：输入已有凭证
```

### OAuth 连接弹窗

```
┌─── 连接小红书账号 ──────────────────────┐
│                                         │
│  1. 在新窗口中授权盎然内容访问你的账号     │
│  2. 授权成功后自动跳回此页面              │
│                                         │
│  ┌──────────────────────────────────────┐│
│  │  [ 前往小红书开放平台授权 → ]         ││
│  └──────────────────────────────────────┘│
│                                         │
│  或手动输入：                            │
│  ┌────────────────┐ ┌──────────────────┐ │
│  │ App ID         │ │ App Secret       │ │
│  └────────────────┘ └──────────────────┘ │
│                                         │
│  状态：◉ 已连接 · Mira Chen 的小红书      │
│       （授权于 2026-05-20）              │
│                                         │
│  ┌──────┐ ┌──────┐                     │
│  │ 断开  │ │ 完成  │                     │
│  └──────┘ └──────┘                     │
└─────────────────────────────────────────┘
```

### 状态指示

| 状态 | 展示 |
|------|------|
| 未连接 | 灰色 `○ 未连接`，可点击 |
| 连接中 | spinner + "授权中..." |
| 已连接 | 绿色 `● 已连接` + 账号名 + 授权日期 |
| 已过期 | 橙色 `● 授权过期` + 重新连接按钮 |
| 连接失败 | 红色 `● 连接失败` + 错误原因 + 重试 |

## 3. 发布状态机 UI

### 状态流转

```
Draft ─→ Scheduled ─→ Publishing ─→ Completed
                        │  └─ PartialFail ─→ Retry ─→ Publishing
                        └─ Failed ───────→ Retry ─→ Publishing
```

## 4. 失败重试流程

### 失败类型

| 类型 | 原因 | 用户操作 |
|------|------|---------|
| Auth 失败 | token 过期 / 权限不足 | 重新授权 → 重试 |
| 格式错误 | 内容不满足平台要求 | 修改内容 → 重试 |
| 网络错误 | API 超时 / 服务不可用 | 稍后重试 |
| 限流 | API 频率限制 | 自动排队，稍后重试 |

### 重试对话框

```
┌─── 发布失败 ─────────────────────────────┐
│                                          │
│  小红书 — 发布失败                       │
│  原因：access_token 已过期                │
│                                          │
│  [ 重新授权 ]  [ 稍后重试 ]  [ 取消 ]    │
│                                          │
│  TikTok — ✅ 发布成功                     │
│  Facebook — ⏳ 排队中                     │
└──────────────────────────────────────────┘
```

## 5. 各平台格式预览

### 小红书（xhs）

| 字段 | 约束 | 说明 |
|------|------|------|
| 标题 | ≤ 20 字 | 可选，显示在封面下方 |
| 正文 | ≤ 1000 字 | 支持 emoji、分段、#标签 |
| 图片 | ≤ 18 张 | 1:1 / 3:4 / 4:3 |
| 视频 | ≤ 15 分钟 | 可选 |
| 标签 | ≤ 10 个 | 正文中的 #tag |

### YouTube（yt）

| 字段 | 约束 |
|------|------|
| 标题 | ≤ 100 字符 |
| 描述 | ≤ 5000 字符 |
| 标签 | ≤ 500 字符 |
| 缩略图 | 1280×720 px |
| 可见性 | 公开 / 不公开 / 私密 |

### TikTok（tt）

| 字段 | 约束 |
|------|------|
| 视频 | ≤ 10 分钟，≤ 287 MB |
| 描述 | ≤ 2200 字符 |
| 标签 | ≤ 5 个 |

### Facebook（fb）

| 字段 | 约束 |
|------|------|
| 正文 | ≤ 63206 字符 |
| 图片 | ≤ 8 张 |
| 链接预览 | 自动抓取 OG 标签 |
